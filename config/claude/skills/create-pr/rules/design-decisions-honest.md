# Design decisions は実際に検討した案だけを書く

Design decisions の「却下」行には、**自分が本当に検討して比較した上で却下した案** だけを書く。テーブルを埋めるためにそれっぽい案を後付けで足してはいけない。

特に次のものを「却下案」として並べてはいけない:

- タスク指示や既存ルールで **最初から禁止されていた anti-pattern** (検討の余地なく避けたもの)
- 採用案を引き立てるためにでっち上げた、誰もまともに検討していない比較対象

これらを「却下」として並べるのは、設計判断を捏造して水増ししているのと同じ。レビュアーは「比較した結果こうなった」という前提で読むので、捏造があると判断の信頼性が崩れる。

判定基準: その「却下案」を採用案と同列に並べたとき、自分は本当にその選択肢を採るか迷ったか? 迷っていない (最初から無いと分かっていた) なら書かない。

書いた却下案がない判断軸であれば、テーブルにせず採用案を箇条書きで述べるか、判断軸ごとセクションを削る。比較表という形式に合わせるために案を増やす必要はない。
