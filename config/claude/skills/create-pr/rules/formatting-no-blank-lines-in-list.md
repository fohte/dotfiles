# 箇条書き項目の中に空行を挟まない

箇条書きの 1 項目の中に空行を入れない。空行を入れると Markdown レンダリング時にその項目が "loose list" 扱いとなり、各項目が `<p>` タグで囲まれる。結果として項目同士の間隔が広がり、箇条書きとしての一覧性が失われる。

子要素 (引用ブロック・コードブロック・サブバレット) を続ける場合は、親項目の直後に空行なしでインデントして置く。

## ❌ 空行を挟む

```markdown
- 親項目の説明

    > 引用ブロック
    > [出典](URL)

- 次の項目
```

## ✅ 空行を挟まない

```markdown
- 親項目の説明
    > 引用ブロック
    > [出典](URL)
- 次の項目
```

## コードブロックの場合も同様

````markdown
- 親項目の説明
    ```yaml
    key: value
    ```
- 次の項目
````

## 段落を分けたいときは

「同じ項目の中で段落を分けたい」と感じたら、それは項目を分ける合図。別の箇条書き項目に切り出すか、項目の外に段落として書く。
