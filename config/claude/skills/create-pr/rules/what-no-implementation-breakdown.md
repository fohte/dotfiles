# 実装の内訳を並べない

diff を見ればわかる内訳をコロン区切り・括弧書き・サブバレットで付け足さない。「1 つの変更を構成要素に分解して列挙する」行為は全て内訳にあたる (ファイル名、UI パーツ、テストカテゴリ、設定キー、件数・数量など)。

- ❌ `プロジェクトの雛形を作成する: app.gemspec, Gemfile, Rakefile, lib/app.rb` (ファイル名の内訳)
- ❌ `CI を追加する: v3.1〜4.0 のテストマトリクス、linter による静的解析` (ツール名・設定値の内訳)
- ❌ `E2E テストを追加する` + サブバレットでテストカテゴリの内訳
- ❌ `E2E テストを 38 件追加する` (件数は diff でわかる)
- ✅ `プロジェクトの雛形を作成する`
- ✅ `CI/CD ワークフローを追加する`
- ✅ `E2E テストを追加する`
