# 根拠はリンク + 引用文をセットで記載

## 参考リンクの事前確認

タスクの指示、委任タスク、issue に参考リンク (PR URL、issue URL など) が記載されている場合、ドラフトを書き始める前に必ずそのリンク先の内容を読むこと。

## from: 表記

この PR の起点となった issue や PR がある場合、Why の最初の行に `- from: <URL>` で記載する。複数ある場合はカンマ区切り。`from:` に入れるのは起点リンクのみ。参考資料や根拠のリンクは説明文の中に埋め込む。

委任タスクや issue の指示に「先行 PR」「参考 PR」「直前の経緯」として PR/issue URL が提示されている場合、それは原則として **この PR の起点** であり `from:` に書く対象。タスクの背景・モチベーション・直前の経緯セクションに登場するリンクをドラフトに転記しないまま submit するのは、レビュアーが文脈をたどれなくなる典型的な失敗パターン。「参考」と書かれていても、そこから話が続いている PR ならば `from:` に入れる。

## リンクの埋め込み方

リンク先を読まなくても意味がわかるように、説明文の中にリンクを配置する。

- ❌ `- 参考: https://github.com/foo/bar/pull/456` (リンクだけで何の参考かわからない)
- ✅ `- 別リポジトリではこの問題が修正済み (https://github.com/foo/bar/pull/3)`

## 引用の書式

根拠は引用文とリンクをセットで記載する。リンク先が変更されても追跡可能にする。引用は原文をそのまま使用し、勝手に要約・省略・変更してはならない。

```markdown
> 引用文 (原文をそのままコピー)
> [ドキュメント名](URL)
```
