# Behavior Review Reference

最初に `references/_common.md` を読み、動作原則・禁止事項・出力形式ボイラープレートを適用する。本ファイルは behavior グループ固有の差分のみを記述する。

## 担当観点

各観点について「この diff でこの軸は壊れうるか」を必ず一度問い、結論を出力に残す。

1. **正しさ (Correctness)**: 仕様通りに動くか。境界条件・エッジケース・入力範囲外で壊れないか。
2. **セキュリティ**: 認証・認可・入力検証・機密情報漏洩・各種 injection。
3. **パフォーマンス**: 計算量・メモリ・I/O・N+1・不要な処理。
4. **並行性 (Concurrency)**: race condition・deadlock・atomicity・順序保証。
5. **エラーハンドリング**: 失敗経路・部分失敗時の状態整合性・cleanup・retry 戦略。

## 出力形式

```
## 観点別評価 (behavior)

1. 正しさ: <マーカー> <一行>
2. セキュリティ: <マーカー> <一行>
3. パフォーマンス: <マーカー> <一行>
4. 並行性: <マーカー> <一行>
5. エラーハンドリング: <マーカー> <一行>

## 指摘詳細 (behavior)

(_common.md のテンプレに従う。ゼロ件なら "指摘なし" の 1 行)
```
